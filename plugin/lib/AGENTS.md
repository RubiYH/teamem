# lib

## Purpose

Prebuilt JavaScript bundles for runtime deployment. These are production artifacts committed to the repository (built via `bun build` commands from `src/`). Every bundled artifact is self-contained and requires no npm install; they carry their dependencies inline.

## Key Files

| File | Description |
|------|-------------|
| `bridge.js` | MCP stdio server + HTTP client bridge; compiled from `src/bridge/index.ts`; invoked by slash commands and hooks in argv mode |
| `channel.js` | Channels POC runtime; compiled from `src/channel/index.ts`; optional MCP server for real-time discussion delivery |
| `setup.js` | Interactive setup wizard; compiled from `src/cli/setup.ts`; invoked by `/teamem:setup` and the teamem-onboarding skill |

## For AI Agents

### Working In This Directory

- **Committed bundles**: These `.js` files are committed to the repository as part of the plugin distribution. They are NOT generated by the user's install process — they ship with the plugin.
- **Freshness contract**: After any change to `src/bridge/`, `src/cli/setup.ts`, or `src/channel/`, you MUST rebuild the committed plugin bundles:
  ```bash
  bun run build:plugin
  ```
  For a channel-only change, `bun run build:channel` is sufficient.
  The test suite asserts byte-equivalence between source and bundles (`tests/plugin/bundle-freshness.test.ts`).
- **Version lock**: When bundled artifacts change, bump the version in `plugin/.claude-plugin/plugin.json`. Claude Code's cache integrity check silently rejects plugins with mismatched versions.

### Common Patterns

- **bridge.js invocation**: Used by all slash commands and hooks. Modes:
  - `bun run bridge.js call <tool> [args...]` — blocking tool invocation (return JSON)
  - `bun run bridge.js poll [--cursor ...]` — long-poll for team events (used by monitor)
  - CLI mode with `--space`, `--actor`, `--token-budget`, `--json '<payload>'` argument flags
- **setup.js invocation**: Used by `/teamem:setup` slash command and the teamem-onboarding skill. Prompts user for space name, member name, and server URL. Returns JSON with created/joined space details.
- **channel.js invocation**: Optional Channels POC runtime. Implements MCP server for real-time discussion delivery via Claude Code's custom channels. Disabled by default; enabled with `--dangerously-load-development-channels server:teamem-channel`.

### Bundle Size and Dependencies

- **bridge.js** (~800 KB): Contains full MCP SDK, Hono, Zod, ULID, and all server logic inlined
- **setup.js** (~65 KB): Minimal bundle with input validation only
- **channel.js** (~800 KB): Similar size to bridge.js; contains MCP server runtime

All three are self-contained with zero npm dependency at runtime (Bun provides the interpreter and stdlib).

## Dependencies

### Internal

- `src/bridge/index.ts` (bridge entry point, tool bindings, HTTP client)
- `src/cli/setup.ts` (interactive setup CLI)
- `src/channel/index.ts` (Channels POC MCP server)
- `src/domain/` (domain logic, pure functions)
- `src/infra/` (database access, validation)

### External

- `bun` (build tool and runtime; user must have Bun installed)
- No npm packages required at runtime (all dependencies bundled inline)
